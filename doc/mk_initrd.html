mk_initrd

mk_initrd creates an initial ram disk initrd.

The files to go onto the ram disk (aka initrd) are taken from data/initrd/initrd.file_list.

The initrd uses a Minix file system. The actual fs size and inode numbers are coded in the mk_initrd script. You can vary these values but it doesn't really pay of much. You will gain or loose a few bytes of the compressed initrd size only.

The linuxrc binary is taken from the location given by the linuxrc entry in the etc/config file.

The final compressed image is stored as images/initrd.
