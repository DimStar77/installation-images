mk_boot

mk_boot creates a boot disk.

It packs the files from data/boot/boot.file_list into a 1.44M FAT image and stores the image file as images/boot.

It needs syslinux to do it. The location of the syslinux binary is given in the syslinux entry in etc/config. Normally something like

syslinux=../syslinux/syslinux

(if you are within the CVS tree).

In addition to making a boot image, mk_boot writes the maximum allowed size for initrd to tmp/boot.max_initrd_size. This is intended to be potentially useful for the mk_initrd script (but currently this info isn't used).

mk_boot creates a FAT file system with only one FAT and with 15 root directory entries. This saves 11 kbytes on a 1.44M floppy.

Note: if by any chance your initrd is just too big by a few bytes, you can try varying the cluster size. It may help.
